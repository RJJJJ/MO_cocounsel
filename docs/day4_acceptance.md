# Day 4 Acceptance - Result List Probe

## today objective
Build a probe parser that inspects the saved Day 3 HTML response and determines whether it is a real search result page, while extracting a first draft of result items.

## deliverables
- `crawler/parsers/court_result_list_probe.py`
- `crawler/parsers/court_result_list_spec.md`
- `data/parsed/court_probe/result_items_last_30_days.json` (generated by probe run)
- `data/parsed/court_probe/result_items_probe_report.txt` (generated by probe run)
- `docs/day4_acceptance.md`

## acceptance checklist
- [ ] Uses BeautifulSoup only (no Playwright).
- [ ] Reads input artifact from `data/raw/court_probe/search_attempt_last_30_days.html`.
- [ ] Performs exploratory parsing (no single hard-coded result selector assumption).
- [ ] Scans possible list structures including rows/cards/repeated blocks and link-centric clues.
- [ ] Attempts extraction fields:
  - [ ] `raw_text`
  - [ ] `title`
  - [ ] `case_number`
  - [ ] `judgment_date`
  - [ ] `court`
  - [ ] `detail_url`
  - [ ] `document_url`
- [ ] Writes JSON output to `data/parsed/court_probe/result_items_last_30_days.json`.
- [ ] Writes diagnostic report to `data/parsed/court_probe/result_items_probe_report.txt`.
- [ ] Terminal output includes:
  - [ ] total candidate links found
  - [ ] total repeated blocks found
  - [ ] total result items extracted
  - [ ] whether page looks like real result page
  - [ ] whether page looks like same search form page returned again
- [ ] Includes basic error handling with explicit failure output.

## evidence developer must provide
- Command used to run probe.
- Terminal output containing the required five summary lines.
- Snippet (or file proof) that JSON output was generated.
- Snippet (or file proof) that report output was generated.
- Short conclusion: current HTML appears to be result page vs form-return page.
