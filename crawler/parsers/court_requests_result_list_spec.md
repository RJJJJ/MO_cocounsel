# Court Requests Result List Spec (Day 8)

## Input artifact
- Primary input HTML:
  - `data/raw/court_probe/requests_replay_after_submit.html`
- Context:
  - Generated by Day 7 requests replay flow after submit.
  - Final URL expected: `https://www.court.gov.mo/zh/subpage/searchresult`.

## Parsing strategy
- Parser implementation: `crawler/parsers/court_requests_result_list_parser.py`.
- HTML parser: BeautifulSoup (`html.parser`).
- Probe-style exploratory extraction (no rigid single selector assumption):
  1. Scan broad container candidates: `tr`, `li`, `div`, `section`, `article`.
  2. Evaluate each block with result-like heuristics:
     - case-number pattern,
     - date pattern + links,
     - or sufficiently rich text + links.
  3. Track structure buckets (`table`, `list`, `card`) and repeated card signatures.
  4. Parse candidate blocks into item objects.
  5. Deduplicate by `(title, case_number, raw_text)`.

## Observed structure type
- Determined dynamically at runtime using candidate counts:
  - `table` if table rows dominate,
  - `list` if list items dominate,
  - `card` if repeated card-like blocks dominate,
  - `mixed` when multiple structures are present.
- Current run-time decision is written to report as `structure_guess`.

## Extracted fields
Per result item, parser attempts to extract:
- `raw_text`
- `title`
- `case_number`
- `judgment_date`
- `court`
- `detail_url`
- `document_url`

Outputs:
- JSON items: `data/parsed/court_probe/requests_result_items.json`
- Report: `data/parsed/court_probe/requests_result_items_report.txt`

## Extraction quality summary
Report includes:
- total candidate blocks found
- total result items extracted
- structure guess (`table/list/card/mixed/unknown`)
- number of items with case number
- number of items with date
- number of items with detail link
- sample items (first 5)

## Missing/uncertain fields
- When result HTML shape changes, title/detail/document links may be ambiguous.
- Court extraction is marker-based and may under-capture full court names.
- Case/date regex may need extension for edge formats.

## Recommended next step
- Preferred next step:
  - Parse detail pages from `detail_url` to collect canonical metadata and full content.
- Alternative if list quality is low:
  - Refine list parser heuristics/selectors with additional replay snapshots.
